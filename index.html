// features

Everything you need to post clean release notes

📝
Simple input, clean output

Fill in your version, features, and fixes through a guided command. Released formats it into a tidy, readable post — no Markdown wrangling required.

📎
.md file included

Every post comes with a downloadable .md attachment, so your release notes are archived in a clean, portable format from day one.

💬
Posts right in Discord

No dashboard to check, no separate changelog site to maintain. Your release notes show up exactly where your community already is.

🏷️
Tag your changes

Mark each line as a new feature, an improvement, or a fix. Readers get instant visual context for what actually changed.

⚙️
Pick your channel

Choose exactly which channel each release gets posted to — keep announcements separate from general chat without any extra setup.

🪶
Lightweight by design

No integrations to configure, no repo access required. Just you, the command, and a release note that looks like you meant it.

// how it works

Up and running in minutes

01 —
Add the bot

Invite Released to your Discord server with one click. No permissions beyond posting in the channels you choose.

02 —
Run the command

Use a simple slash command to enter your version number and list out what's new, improved, or fixed.

03 —
Pick a channel

Tell Released where to post — your #announcements or #release-notes channel, or anywhere else you'd like.

04 —
It's posted

Released formats everything into a clean message with a .md attachment, ready for your community to read.
